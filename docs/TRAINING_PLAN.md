# Team Training Plan

Audience: Communications team members with mixed technical backgrounds.

## Pre-Requisites

- Slack access (for receiving digests and alerts)
- Familiarity with current manual workflows (clip curation, rapid response playbook)
- No coding experience required

## Session 1: Understanding the Tools (45 min)

### Objectives
- Understand what each workflow does and does NOT do
- Know how to read confidence scores and priority tiers
- Know when to override the system and escalate manually

### Agenda
1. **Demo: Press Digest** (15 min)
   - Walk through a sample digest output
   - Explain relevance scoring (1-10 scale, threshold at 6)
   - Show how topic and sentiment tags work
   - Discuss the "rationale" field — Claude explains every decision
2. **Demo: Rapid Response** (15 min)
   - Walk through sample alerts at each tier (P0, P1, P2)
   - Explain SLA expectations per tier
   - Review auto-generated talking points vs. human judgment
   - Practice: would you agree with this tier assignment?
3. **Demo: Briefing Generator** (10 min)
   - Show input request format
   - Walk through a generated briefing
   - Identify sections that need human review vs. ready-to-use
4. **Q&A** (5 min)

## Session 2: Hands-On Workflow Drills (60 min)

### Objectives
- Run each workflow independently
- Practice reviewing and editing AI-generated outputs
- Build confidence in overriding or supplementing AI recommendations

### Exercises
1. **Digest Review Drill** (20 min)
   - Receive a sample digest in Slack
   - Identify one article that should have been included but wasn't
   - Identify one article that was included but shouldn't have been
   - Discuss: how would you adjust the relevance threshold?
2. **Rapid Response Tabletop** (25 min)
   - Receive 3 simulated events, each with Claude's assessment
   - For each: agree/disagree with tier? Edit talking points? Change routing?
   - Practice drafting a response using the generated talking points as a starting point
3. **Briefing Review** (15 min)
   - Generate a briefing for an upcoming (simulated) interview
   - Edit the "anticipated questions" section based on your domain knowledge
   - Identify one question Claude missed that you'd add

## Session 3: Advanced Usage & Feedback (45 min)

### Objectives
- Understand how feedback improves the system
- Learn to request custom configurations
- Feel ownership over the tooling

### Agenda
1. **Feedback Loops** (15 min): How your corrections improve future outputs
2. **Custom Configurations** (15 min): Adjusting thresholds, sources, routing
3. **Open Workshop** (15 min): Bring your own workflow challenge

## Reinforcement Plan

- **Weeks 1-4**: Weekly 30-min office hours
- **Day 14**: Adoption survey (ease of use, trust, time savings)
- **Day 30**: Impact survey (workflow changes, satisfaction)
- **Ongoing**: Shared FAQ doc updated after every office hours session
- **Monthly**: Review eval metrics with team leads
